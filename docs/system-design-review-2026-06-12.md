# 系统功能与设计文档核查 - 2026-06-12

## 1. 核查结论

当前项目仍应定位为早期 alpha，不适合描述为生产可用系统。

代码实现已经从最初的 PHP/MariaDB 骨架推进到可演示的制造 ERP 基础闭环，已覆盖登录、仪表盘、物料、仓库、库存、BOM、采购单、生产工单、缺料分析、批次追溯、用户与角色管理、CLI、迁移、测试和 GitHub Actions。

文档总体诚实，没有夸大用户、下载量、生产成熟度等信息。但部分设计文档是历史记录，内容已经落后于当前实现，需要在后续开发中分清“原始愿景”“已完成计划”“当前设计基线”和“下一步详细设计”。

## 2. 当前已实现功能

### 2.1 系统基础

- PHP 8.3 纯 PHP 应用基础。
- MariaDB 兼容迁移脚本。
- 轻量自动加载、路由、配置读取。
- Web 入口 `public/index.php`。
- CLI 入口 `bin/erpctl`。
- CLI 命令：健康检查、迁移、创建管理员。
- 共享主机部署包构建脚本。
- 自定义 PHP 测试运行器。
- GitHub Actions 测试工作流。

### 2.2 登录、安全与权限

- 管理员登录。
- Session 登录态。
- CSRF 校验。
- 未登录访问受保护页面时跳转登录。
- 用户管理页面。
- 用户创建、启用、停用。
- 角色选择。
- 当前角色：
  - 管理员 `admin`
  - 总经理 `general_manager`
  - 主管 `supervisor`
  - 计划员 `planner`
  - 仓库员 `warehouse`
  - 采购员 `purchasing`
- 已有关键操作权限检查，例如采购、收货、工单、领料、完工、库存调整、缺料查看、用户管理。

### 2.3 基础资料

- 物料档案列表、创建、搜索、编辑、启用、停用。
- 仓库档案列表、创建、搜索、编辑、启用、停用。
- 客户档案列表、创建、搜索、启用、停用。

### 2.4 销售

- 销售订单创建。
- 销售订单关联客户档案和成品物料。
- 销售订单记录数量、交付日期和状态。
- 销售订单可基于有效 BOM 手动生成生产工单草稿。

### 2.5 库存与追溯

- 库存事务录入。
- 入库、出库、调整。
- 按物料和仓库计算当前库存。
- 库存余额页面。
- 批次追溯页面。
- 仪表盘库存指标。

### 2.6 BOM

- 按项目维护单层 BOM，支持项目编号和项目名称。
- 单层 BOM 创建。
- BOM 组件维护。
- BOM 搜索、启用和停用。
- 按计划数量计算物料需求。
- 防止父件作为自身组件。

### 2.7 采购

- 供应商档案列表、创建、搜索、启用、停用。
- 采购单创建。
- 采购单可关联供应商档案。
- 采购金额计算。
- 采购收货入库存。
- 防止重复收货。
- 已有采购与收货权限检查。

### 2.8 生产工单

- 根据项目 BOM 创建生产工单。
- 停用的项目 BOM 不允许用于新建生产工单。
- 生成工单物料需求。
- 工单领料出库。
- 完工成品入库。
- 防止重复领料和重复完工。
- 已有工单创建、领料、完工权限检查。

### 2.9 计划与缺料

- 对计划状态工单做基础缺料分析。
- 按 BOM 需求和当前库存计算缺料。
- 缺料页面受权限控制。
- 根据缺料分析生成采购建议。
- 授权采购用户可以将采购建议转为采购订单草稿。

## 3. 当前未完成能力

### 3.1 业务闭环缺口

- 销售订单已支持手动生成生产工单，但尚未实现自动排产、交付跟踪和销售状态闭环。
- 采购单已支持部分收货和退货，但缺少审批、关闭等完整状态流。
- 工单缺少退料、补料、部分完工、报工、关闭等完整状态流。
- BOM 仍是项目级单层基础能力，尚未实现多级展开、版本生效日期、审批、替代料。
- 成本可视化尚未实现。

### 3.2 管理与合规缺口

- 操作审计日志尚未实现。
- 数据范围权限尚未实现，例如按仓库、部门、业务范围隔离数据。
- 用户密码策略仍偏开发环境，后续生产版本需要更强策略。
- 安装向导、升级、备份、恢复尚未实现。

### 3.3 使用体验缺口

- 只有物料使用说明文档，其他模块缺少用户操作手册。
- Excel 导入导出尚未实现。
- 打印、导出、报表视图尚未实现。
- 移动端列表体验仍需针对表格较多的页面优化。
- 各模块空状态、错误提示、下一步引导还可以继续增强。
